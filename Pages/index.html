Flexbox

This week I spent some time working with flexbox/(flexible box) this is a CSS layout module that provides an efficient way to distribute space and align items in a container. even when their size is dynamic. The main reason to use Flexbox is to create responsive and flexible layouts that adapt to different screen sizes and orientations. This works well better than inline or block which work horizontally and vertically.

CSS

I also spent time exploring CSS design in more detail. I used div elements to divide the webpage into sections and applied CSS to control the layout and spacing of those sections. By doing this I started building reusable layout components that can be used across multiple pages. This helped me better understand how larger websites keep their design consistent.

Terminology

  • Parent: The container that holds the flex items. It is defined by setting the display property to flex or inline-flex.
  • Child: The items that are contained within the flex container. They can be any HTML elements, such as divs, images, or text
  • Display: sets the type of layout for the container. such as Flexbox, Inline, Block.
  • Flex-direction: defines the direction of the flex items in the container. such as row, column, row-reverse, column-reverse.
  • Justify-content: defines how the flex items are aligned along the main axis. such as flex-start, flex-end, center, space-between, space-around.
  • Align-items: defines how the flex items are aligned along the cross axis. such as flex-start, flex-end, center, stretch.
